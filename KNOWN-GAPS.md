## AI / connector effector layer (opt-in, off the deploy critical path)

The connector framework is an agent **effector layer** (invoke/scope/approval + budgeted autonomy + delegation-ID + decision-class). It is fully unit/integration tested, but the following are validated against test doubles rather than the live world:

- **Connectors tested against stubs, not live external APIs** (severity: info — the largest current gap). The catalogue is **42 providers** (the original 4 + 38 commercial-lane providers built 2026-06-13: WhatsApp, Shopify, Stripe, HubSpot, Zendesk, SendGrid, Twilio, Telegram, Teams/Google-Chat/Zapier/Make/n8n webhooks, Gupshup/Plivo/Exotel/Kaleyra/Sinch, Cashfree, Freshdesk, Intercom, Pipedrive, Freshsales, Mailchimp, WooCommerce, Typeform, Jotform, ServiceNow, Jira, monday, Asana, ClickUp, Trello, Notion, Airtable, ActiveCampaign, Klaviyo, Mailgun). Every provider has a unit test, but each is implemented against the **documented** API shape (endpoint, auth header, body) and stubbed at `Net::HTTP` — **none has been exercised against a live endpoint with real credentials.** Auth models + endpoints were chosen from official docs, but verify each against the live API (and re-check time-sensitive details, e.g. WhatsApp pricing, Shopify REST→GraphQL) before relying on it in production.
- **OAuth2 connectors not yet built — no token-refresh seam** (severity: info, deferred). The 42 providers are all static-credential (API key / token / Basic / webhook). The remaining ~30 roadmap connectors (Salesforce, Zoho, Dynamics, the Google Workspace suite, Microsoft 365 / Teams / SharePoint, Marketo, DocuSign, Box/Dropbox/Drive, Zoom…) need an **OAuth2 authorization-code + token-refresh seam** on the credential vault plus an admin connect/callback flow — a foundation not yet built. Gov DPI connectors remain licence-gated (see the connector-lane decision).
- **Agent dispatch loop run only with the `fake` LLM client** (severity: info). `Connectors::AgentRunner` + `Llm::HttpClient#chat_with_tools` are driven by `Llm::FakeClient` in tests; the end-to-end "tool-use turn → `Connectors::Invoke.call`" loop has not been run against a real model endpoint.
- **No OAuth2 token-refresh seam on the credential vault** (severity: info, deferred). Current connectors use static credentials / webhooks. An OAuth2 refresh seam must be designed before the first OAuth provider (Salesforce / HubSpot / M365 / Google).
- **Decision-class doctrine is judicially untested for AI in India** (severity: info). The autonomous / confirm / of_record gating is grounded in administrative-law principles (reasoned-order duty, non-fettering, audi alteram partem), not in case law applying them to AI-influenced decisions — monitor (e.g. SC Draft AI Regs for Courts 2026).
